# 데이터 탐색 — 포트폴리오 · 시나리오 · 리뷰 센터 · 근거 탐색

위험 지도와 변경 영향이 "질문에 답하는" 층이라면, 데이터 탐색 그룹은
**답의 근거가 사는 곳**입니다. 상단 내비게이션의 "데이터 탐색" 그룹에서 진입합니다.

## 포트폴리오

프로젝트 U/V/W의 전반 상태를 한 화면에 요약합니다.

- **프로젝트 요약 카드** — 마일스톤/진행 중 요청/이벤트 수.
- **주의 항목 (lane 6종)** — 검토가 필요한 항목을 종류별로 묶습니다:

| Lane | 의미 |
|---|---|
| 근거 부족 | 요청에 필요한 근거가 누락됨 |
| 정의 필요 | 그룹이 참조하지만 아직 정의되지 않은 시나리오 |
| 확신도 차단 | 근거 미가용으로 판단 확신도에 상한이 걸림 |
| 전파 검토 | 프로젝트 간 전파 기록 (인과 증명 아님 — 검토 연관) |
| 리스크 해소 후보 | 검토 대기 중인 후보 옵션 |
| 경영 주의 | P1 + 경영 관심사가 명시된 요청 |

  각 lane은 상위 5건만 펼쳐져 있습니다 — **더 보기**로 전체를 확인하세요.
  항목의 "검토 참여 권장 역할"은 참여 권장이며 담당 지정이 아닙니다.

- **시나리오 × 프로젝트 매트릭스** — 시나리오별 요청/이벤트/근거 공백 수. 카드를 클릭하면
  시나리오 상세로 이동합니다.

## 시나리오 목록 / 시나리오 상세

시나리오 상세는 이 시스템의 **최종 drill-down 지점**입니다. 위험 지도의 근거 패널,
변경 영향의 시나리오 링크, 포트폴리오 매트릭스가 모두 여기로 모입니다. 탭 구성:

| 탭 | 내용 |
|---|---|
| 개요 | 기본 정보 · **근거 공백**(누락/미가용/요구 미충족/확신도 차단) · KPI · 사용 IP · 요청 · 이슈 · 변형 · 측정 |
| 타임라인 | 이벤트/활동/요청/마일스톤을 주차(W) 순으로 |
| 이벤트·활동 | 개발 이벤트와 역할 활동 — 활동의 우려/권고는 근거 문장(도출 과정 포함) 형식 |
| 조언 | LLM 기반 역할별 조언 생성 (아래 참조) |
| 추적 | Traceability — 이 객체를 참조하는/이 객체가 참조하는 모든 연결 |

### 조언 탭 — LLM 조언을 읽는 법

**조언 생성**을 누르면 7개 역할 관점의 조언이 생성됩니다. 해석 시 알아둘 것:

- **생성 엔진 뱃지** — Claude(외부 LLM) / 사내 LLM / **결정론 규칙**(LLM 미가용 시 fallback).
  어느 엔진이든 evidence-grounded 검증 관문을 통과한 출력만 표시됩니다.
- **확신도** — low/medium/high. 근거가 부족하면 시스템이 high를 허용하지 않습니다.
- 각 조언에는 **뒷받침 근거**(온톨로지 객체 ID)와 **도출 과정**이 붙습니다.
- 모든 조언은 "최종 결정이 아닌 검토 조언"입니다.

### 추적 탭 — Traceability

객체 간 연결을 양방향으로 따라갑니다 (이 객체가 참조 / 이 객체를 참조).
breadcrumb으로 경로를 유지하며 연쇄 탐색할 수 있습니다. 표시되는 관계는
**인과 증명이 아닌 연결 관계**입니다.

## 리뷰 센터

주차(W)를 선택하면 그 주의 개발 이벤트 / 역할 활동 / 시나리오 요청 스냅샷을 보여줍니다.
주간 리뷰 미팅의 준비 화면으로 사용합니다.

**결정의 "당시 상태 보기"**: 반입된 결정에는 **데이터 시점** 칩(시스템이 그 결정을
알게 된 시각)이 붙습니다.

- **🕐 당시 위험 지도** — 그 시점의 위험 지도를 재구성해 엽니다: "이 결정은 어떤
  그림을 보고 내려졌나"를 다시 봅니다.
- **⇄ 당시↔현재 비교** — 결정 이후 어떤 셀이 바뀌었는지 변화만 모아 보여줍니다:
  "결정의 전제가 아직 유효한가"를 점검합니다.
- 버전 기록이 시작되기 전의 결정(가상 시드 등)에는 **캡처 이전 결정** 배지가 붙고
  링크가 없습니다 — 재생할 수 없는 시점을 재생하는 척하지 않습니다.

## 근거 탐색

근거 카탈로그 전체를 프로젝트/가용성으로 필터링합니다.

- **가용성 뱃지** — available(초록) / partial(노랑) / missing(빨강) / planned(파랑).
- **출처 뱃지** — 가상(synthetic 시드) / 반입(Excel·CSV로 들어온 실데이터) / 연동(커넥터).
  반입 이력 카드에서 배치별 수용/거부 건수를 확인할 수 있습니다.
- 측정/예측 구분 뱃지 — 측정된 근거인지 예측치인지.

데이터는 반입(ingest) 경로로만 들어오고, 삭제는 반입 배치 단위 롤백만 가능합니다 —
화면에서 개별 데이터를 수정/삭제하는 기능은 의도적으로 없습니다.
