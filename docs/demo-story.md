# 데모 스토리 — 4장면 진행 가이드

> 흐름: **위험 발견 → 원인 분석 → 변경 영향 → 결정 근거**

내비게이션 오른쪽 끝의 **데모 스토리** 버튼을 누르면 시작됩니다. 각 장면은 사전
구성된 화면으로 자동 이동하며, 화면 상단 배너의 **다음 장면** 버튼만으로 진행됩니다.
장면별 소요 시간이 자동 기록되어 마지막에 TAT 요약이 표시됩니다.

이 문서는 데모를 **진행하는 사람**을 위한 대본입니다. 각 장면에서 무엇이 보이는지,
어디를 클릭하는지, 어떤 메시지를 전달하면 되는지를 정리했습니다.

## 장면 1 — 위험 발견 (위험 지도)

**보이는 것**: Project U 위험 지도. 최상단 붉은 행 = **8K30 Recording KPI**.

**할 것**: 8K30 행의 **MFC 셀**을 클릭 → 판정 근거 패널에 미해결 이슈
"8K30 encode violates bitrate-latency constraint"가 표시됩니다.

**전달 포인트**:

- 진입 10초 안에 "이번 주 어디가 위험한가"가 0클릭으로 식별됩니다 — 행은 위험한
  시나리오부터 정렬됩니다.
- 등급은 수치 점수가 아니라 **룰 기반 정성 판정**이며, 셀을 클릭하면 발화한 룰과
  원본 객체(이슈/요청/이벤트)가 그대로 근거 목록으로 나옵니다.
- 우측 하단 **이번 주 주목** 목록에서 P1 요청 "Project U 8K30 recording thermal
  and power review"(누락 근거 2건)를 짚어두면 장면 4의 복선이 됩니다.

화면 해석 상세: [위험 지도 가이드](risk-map.md)

## 장면 2 — 원인 분석 (이슈 분석 · RCA 체인)

**보이는 것**: 장면 1에서 본 8K30 인코딩 이슈의 RCA 체인
(증상 → 영향 → 원인 → 조치 → 검증 → 잔존 리스크 → 교훈).

**할 것**: **원인 노드**에서 후보 "NAL queue batching"을 확인합니다.
**검증 테스트 노드가 빨간색**임을 짚습니다.

**전달 포인트**:

- 원인은 아직 "후보(미확정)" 단계 — 시스템은 확정된 것과 추정인 것을 구분해 보여줍니다.
- 검증 노드가 빨갛다는 것은 **해결 주장을 확인할 테스트가 없다**는 뜻입니다.
  "고쳤다"는 말과 "고쳐졌음이 확인됐다"를 시스템이 구분합니다.

화면 해석 상세: [이슈 분석 가이드](issues.md)

## 장면 3 — 변경 영향 (개선안의 파급)

**보이는 것**: 장면 2의 원인 후보였던 **MFC NAL Queue mode** 조정의 영향 분석 —
영향 시나리오 3건, KPI 12건, 연쇄 IP, 유사 사례.

**할 것**: 영향 전파 지도에서 시나리오/KPI/연쇄 IP 경로를 따라가고,
역할별 검토 체크리스트를 확인합니다.

**전달 포인트**:

- "이 knob을 바꾸면 어디에 영향이 가나"를 회의 소집 전에 확인할 수 있습니다.
- 결과는 결정이 아니라 **검토 안내**입니다 — 검토에 참여할 역할과 관점을 제안할 뿐,
  담당자 지정이나 수치 점수는 없습니다.

화면 해석 상세: [변경 영향 가이드](change-impact.md)

## 장면 4 — 결정 근거 (Ask SoC)

**보이는 것**: "8K30 thermal issue가 해결됐다고 판단할 evidence는 무엇인가?"에 대한
인용 동반 답변.

**할 것**: 인용 카드에서 **thermal soak 테스트 결과 실패**(10분 목표 전 스로틀링)와
**P1 요청 근거 미확보**를 확인합니다. 각주를 클릭하면 원본 근거로 이동합니다.

**전달 포인트**:

- 이 장면의 결론은 "**아직 해결로 판단할 근거가 없다**"입니다 — 그리고 그 결론
  자체가 인용으로 뒷받침됩니다. 근거 없는 확신을 차단하는 것이 이 시스템의 핵심
  가치입니다.
- 장면 1에서 봤던 P1 thermal 검토 요청이 여기서 다시 등장합니다 — 위험 지도의
  판정과 Ask의 답변이 같은 원본 객체를 참조합니다.

화면 해석 상세: [Ask SoC 가이드](ask.md)

## 진행 팁

- **TAT 요약**: 마지막 장면에서 "다음 장면"을 누르면 장면별 소요 시간과 합계가
  표시됩니다. 데모 후 기준표에 옮겨 적으세요.
- **다시 시작**: 장면 1로 돌아가면(이전 장면 버튼 포함) 시간 기록이 초기화됩니다 —
  재시작 용도입니다.
- **URL 공유**: 모든 장면 상태는 URL에 반영되므로, 특정 장면 주소를 복사해 보내면
  상대방 화면에서 그대로 재현됩니다.
