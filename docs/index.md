# SoC 운영 온톨로지 — UI 가이드

Multimedia SoC 개발 운영 온톨로지는 프로젝트·시나리오·IP·이벤트·근거를 하나의
그래프로 연결하고, 실무 리더의 질문에 **근거와 함께** 답하는 시스템입니다.

이 문서는 화면을 **어떻게 사용하고, 표시된 내용을 어떻게 해석하는지**를 설명합니다.

## 설계 철학 — 질문이 곧 메뉴다

화면은 데이터 종류가 아니라 **질문** 단위로 구성됩니다.

| 질문 | 메뉴 | 상태 | 가이드 |
|---|---|---|---|
| 지금 어떤 시나리오·IP가 위험한가? 근거는? | **위험 지도** (홈) | 사용 가능 | [위험 지도 가이드](risk-map.md) |
| 이 IP/knob을 바꾸면 어디에 영향이 가나? | **변경 영향** | 사용 가능 | [변경 영향 가이드](change-impact.md) |
| 이 이슈의 원인은? 정말 해결됐나? | **이슈 분석** (RCA 체인) | 사용 가능 | [이슈 분석 가이드](issues.md) |
| 과거에 비슷한 문제가 있었나? | **Ask SoC** (자연어 질의) | 사용 가능 | [Ask SoC 가이드](ask.md) |
| 이 시나리오의 전체 상황은? | 데이터 탐색 → 시나리오 상세 | 사용 가능 | [데이터 탐색 가이드](explorer.md) |

원점 문서의 5대 질문 메뉴가 모두 활성화되어 있습니다.

## 세 가지 약속

모든 화면이 지키는 원칙입니다. 자세한 내용은 [공통 개념과 해석 원칙](concepts.md) 참조.

1. **근거 없는 답은 없다** — 모든 위험 등급, 영향 항목, 조언은 원본 객체(요청/이벤트/이슈/근거)
   참조 목록을 동반합니다. 어떤 주장이든 3번의 클릭 안에 원본 근거에 도달할 수 있습니다.
2. **수치 점수가 아니라 정성 등급** — 높음/중간/낮음 판정만 표시하며, 판정에 사용된 룰과
   근거를 함께 보여줍니다. 숨겨진 산식은 없습니다.
3. **결정이 아니라 검토 안내** — 시스템은 담당자를 지정하거나 결정을 내리지 않습니다.
   검토에 참여할 역할과 관점을 제안할 뿐입니다.

| 항목 | 값 |
|---|---|
| 접속 주소 (로컬 기준) | `http://127.0.0.1:5275` |

| 데이터 | 현재 synthetic universe (Project U/V/W) + Excel/CSV 반입 데이터 |
