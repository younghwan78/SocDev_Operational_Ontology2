# 공통 개념과 해석 원칙

모든 화면에 일관되게 적용되는 규칙입니다. 한 번 읽어두면 어느 화면이든 같은 방식으로
해석할 수 있습니다.

## 화면의 3단 구조: 질문 → 답 → 근거

모든 화면은 위에서 아래로 "질문에 대한 답 → 그 답의 근거" 순서로 배치됩니다.
근거 없는 답은 화면에 존재할 수 없습니다 — 등급·영향·조언 어디에든 원본 객체 참조가
붙어 있고, **3번의 클릭 안에** 원본 근거에 도달할 수 있습니다.

## 색과 기호

| 색 | 의미 | 예 |
|---|---|---|
| 빨강 | 위험 / 누락 | 위험 등급 높음(●), 누락 근거, 미해결 이슈 |
| 노랑 | 주의 / 부분 | 위험 등급 중간(◐), partial 가용성, 혼합 방향성 |
| 초록 | 정상 / 검증됨 | 위험 등급 낮음(○), available 근거, 감소(개선) 방향 |
| 파랑 | 중립 정보 | 분류 뱃지, 계획된(planned) 근거 |

같은 색은 어느 화면에서든 같은 의미입니다.

## 내부 ID는 숨겨져 있다

화면에는 한국어/표시명이 보입니다. 원본 객체의 내부 ID(예: `req_uhd60_eis_u_w12`)가
필요하면 **해당 항목에 마우스를 올리세요** — tooltip으로 표시됩니다.
시나리오 상세의 근거 공백 목록 등 상세 패널에서는 ID가 직접 표시됩니다.

## 접기 기본

목록은 상위 3~5건만 펼쳐지고 나머지는 접힙니다. **더 보기 (+N)** 을 누르면 전체가
보입니다. 화면에 보이는 건수가 전부가 아닐 수 있으니 괄호의 총 건수를 확인하세요.

## 근거(Evidence-Grounded) 체계

- **근거 항목** — 모든 판정·영향·조언에 붙는 참조 단위. `룰/유형 뱃지` + 설명 +
  원본 객체 참조로 구성됩니다.
- **확신도 (low / medium / high)** — 판단이 얼마나 근거에 의해 뒷받침되는지.
  근거가 부족하면 시스템이 high 확신도를 **허용하지 않습니다** (검증 관문이 차단).
- **확신도 차단** — 특정 근거가 미가용이라 판단 확신도에 상한이 걸린 상태.
  상한이 low면 심각(높음 위험), medium이면 중간 확신까지는 가능합니다.
- **가정(assumption)** — 근거가 없는 서술은 가정으로 표시되고 확신도가 제한됩니다.

## 데이터 출처

| 뱃지 | 의미 |
|---|---|
| 가상 | synthetic universe 시드 데이터 (Project U/V/W 데모 세계관) |
| 반입 | Excel/CSV로 반입된 실데이터 — 행 단위 계보 추적 |
| 연동 | 외부 시스템 커넥터 유래 (JIRA/Confluence — 예정) |

## 시스템이 하지 않는 것

해석할 때 전제로 삼아야 할 의도적 제약입니다:

- **수치 리스크 점수를 만들지 않습니다** — 정성 등급 + 근거 목록만.
- **담당자를 지정하지 않습니다** — 검토 참여 권장 역할만 제안.
- **결정을 내리지 않습니다** — 모든 출력은 "검토 안내"이며 결정 질문을 드러낼 뿐입니다.
- **화면에서 데이터를 수정/삭제할 수 없습니다** — 데이터는 반입 경로로만 진입.
- **LLM이 조회·판정에 개입하지 않습니다** — 위험 등급/변경 영향/추적은 전부 결정론.
  LLM은 조언 생성에만 쓰이며, 그 출력도 근거 검증 관문을 통과해야 표시됩니다.

## 용어 미니 사전

| 용어 | 의미 |
|---|---|
| 시나리오 | 개발/검증 대상이 되는 동작 단위 (예: UHD60 EIS 녹화) |
| 시나리오 그룹 | 같은 목적/KPI 축을 공유하는 시나리오 묶음 |
| 변형(Variant) | 해상도/fps/토글 등 구체 동작 조건 |
| IP 블록 | ISP/DPU/MFC/ABOX 등 기능 IP와 GPU/NPU 등 연산 IP |
| 시스템 블록 | MIF/NoC/SystemMMU/Power·Clock·Thermal — 시스템 영향 블록 |
| knob | 전력·지연·대역폭·리스크 방향성을 갖는 제어 항목 (드라이버 수준) |
| capability | IP가 지원하는 기능/조건/값 |
| KPI | 방향(높을수록/낮을수록 좋음)과 단위를 갖는 지표 |
| 요청 | 주간 검토 활동을 구동하는 요구 단위 (P0/P1/P2 우선순위) |
| 개발 이벤트 | 마일스톤/이슈/검토/전파 등 개발 과정의 사건 |
| 근거 공백 | 필요하지만 없는/미가용인/요구 미충족인 근거 |
| W (주차) | 프로젝트 기준 주차 — 타임라인/리뷰 센터의 시간 축 |
