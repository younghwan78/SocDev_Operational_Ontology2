# 사내 검증 워크숍 자료 — Fixture 가설 목록 (Stage 12)

> 원점 문서 Phase 0D 대응: synthetic universe의 관계·룰이 **실제 업무와 맞는지**를
> 실무 리더에게 묻는다. 각 가설은 "맞다/아니다/조건부"로 판정받고, 아니면
> fixture/룰을 교정한다 (변경 규율 6단계). 데이터가 아니라 **연결 모델**을 검증하는
> 것이 목적이다 — ingestion 자동화보다 이것이 먼저다.

## A. 온톨로지 관계 가설 (연결 모델)

| # | 가설 | 확인 방법 | 판정 |
|---|---|---|---|
| A1 | 시나리오는 사용 IP(uses)와 의존 시스템 블록(depends)으로 나눠 기록하는 것이 실무 사고방식과 맞다 | 위험 지도 열 구성 리뷰 | |
| A2 | IP 의존 규칙은 "조건" 문구가 핵심이다 (무조건 의존은 실무에 없다) | 변경 영향 연쇄 IP 카드 | |
| A3 | 요청(P0/P1)과 근거 공백의 조합이 주간 검토 우선순위를 실제로 결정한다 | 이번 주 주목 5건 리뷰 | |
| A4 | 이슈의 영향 범위를 시나리오×IP×KPI로 기록하면 충분하다 (누락 축이 없는가?) | RCA 영향 노드 리뷰 | |
| A5 | 검증 테스트는 이슈와 다대다로 연결된다 (한 테스트가 여러 이슈를 검증) | tests fixture 리뷰 | |
