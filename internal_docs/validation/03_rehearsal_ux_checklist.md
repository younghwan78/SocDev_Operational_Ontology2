# 리허설 UX 판정 대본 (설계 25) — "원하는 정보를 얻을 수 있는가"

> 대상: `soc_rehearsal` DB (57 유래 가상 JIRA/Confluence 리플레이, wave 43개 ·
> 이슈 80 · 페이지 88 · 결정 24 — 2026-07-19 리플레이 완료).
> 사용법: 8155 백엔드를 리허설 DSN으로 켜고 5275 프론트에서 각 질문을 수행,
> **판정**(찾음/부분/못 찾음)과 소요(클릭 수/초)를 기입한다.
> **못 찾음 = UX 개선 백로그** — 데이터가 답을 보장하도록 심어져 있으므로
> 못 찾으면 화면 문제다.
>
> 준비: `SOC_ONTOLOGY_DSN=postgresql://soc:soc58local@127.0.0.1:5432/soc_rehearsal`
> 재현: `db-init` → `db-seed` → `rehearsal-replay --dsn <위 DSN>`
> (부분 리플레이 `--waves N`으로 "N번째 wave까지의 상태"도 재현 가능)

판정 기준: **찾음** = 안내 없이 2분 내 기대 답 도달 / **부분** = 도달했으나
경로가 비직관적이거나 해석이 필요 / **못 찾음** = 도달 실패.
